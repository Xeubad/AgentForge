# lessons

这份文件用于沉淀**跨项目可复用**的长期经验、协作偏好和纠错结论。

## 使用原则

- 只记录未来在多个项目里仍然有价值的经验
- 优先记录用户明确纠正过、且可能重复出现的问题
- 不记录项目专属目录结构、代码实现细节或临时任务进度
- 不确定是否应写入时，先问：**脱离当前仓库后，这条信息还有没有价值？**

## 当前已初始化经验

### 1. 默认输出语言
- 默认使用中文输出

### 2. 输出风格
- 默认先给结论，再补必要说明
- 用户要求“直接一点”“少废话”时，收敛到高信噪比表达

### 3. 工作方式
- 复杂任务先规划，再执行
- 修改前先读取相关文件，不凭想象改代码
- 完成前尽量验证；不能验证时明确说明风险

### 4. 判断边界
- 遇到第三方 API、框架能力或外部库用法时，不猜，优先查官方文档或要求更多上下文
- 遇到不确定的业务边界时，不替用户编造规则
- 优先复用现有实现，不随意重复发明组件、工具函数或接口层

## 维护规则

- 新增内容应是“跨项目仍成立”的经验，而不是某一次任务记录
- 如果某条经验被新规则覆盖，应直接更新，而不是保留多份近义规则
- 如果一条经验后来证明只对单个项目有效，应从这里移走，改存项目级记忆
