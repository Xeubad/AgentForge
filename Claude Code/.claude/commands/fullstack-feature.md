# Fullstack Feature

标准全栈新功能开发流程。按步骤执行，每一步决定是否调用对应 Skill。

## 适用场景

- 从零开发一个新功能（前后端都有）
- 中等复杂度，需要设计 + 实现 + 审查全流程

## 前置检查

1. 确认 `tasks/skill-manifest.md` 存在并已审核
2. 确认 `tasks/feature-list.json` 中该功能已记录，状态为 `not_started` 或 `in_progress`
3. 确认标准启动路径可用

## 执行流程

### 步骤 1：需求澄清

**人格**：`aoi`

**动作**：
- 确认功能范围和边界
- 检查 `feature-list.json` 中的验收标准
- 如果有模糊点，先澄清再继续

**技能调用**：
```
Skill(skill="brainstorming")
```
- 触发条件：需求不够清晰、有多个实现方向待选
- 输出：需求文档、功能边界清单

**跳过条件**：需求已经清晰，验收标准明确

---

### 步骤 2：架构设计

**人格**：`nagi`

**动作**：
- 根据需求确定技术方案
- 检查是否影响现有架构（数据库 / API 契约 / 前端状态流）
- 如果有架构变更，记录 ADR

**技能调用**：
```
Skill(skill="architecture-patterns")
```
- 触发条件：涉及新模块、新数据流、跨层协调
- 输出：架构决策、数据流图、模块边界

```
Skill(skill="architecture-decision-records")
```
- 触发条件：做了有 trade-off 的技术选型
- 输出：ADR 文档（选了什么、为什么、放弃了什么）

**跳过条件**：功能在当前架构内可以自然落地，无选型决策

---

### 步骤 3：实现

**人格**：根据模块归属，前端 → `mio`，后端 → `rei`

**动作**：
- 按架构设计实现
- 遵循项目编码规范（`PROJECT_STRUCTURE.md` 第 8 段）
- 优先复用现有组件/工具函数

**技能调用**：
```
Skill(skill="frontend-dev")
```
- 触发条件：mio 实现前端组件 / 页面时
- 用途：组件拆分、设计工程、可访问性检查

```
Skill(skill="architecture-patterns")
```
- 触发条件：rei 实现新接口或涉及数据模型变更时
- 用途：后端架构模式、接口规范、数据流设计

```
Skill(skill="tdd")
```
- 触发条件：**新功能开发默认开启**；跳过条件：纯 bug 修复、临时调试
- 流程：先写测试 → 运行（红）→ 实现 → 运行（绿）→ 重构

**并行规则**：前后端如果接口契约已定，mio 和 rei 可以并行实现

---

### 步骤 4：代码审查

**人格**：`kanon`

**动作**：
- 检查代码品味和一致性
- 确认实现符合架构设计
- 检查是否有遗漏的边界情况

**技能调用**：
```
Skill(skill="code-review")
```
- 触发条件：实现完成，合并前
- 检查项：代码品味、命名一致性、异常处理

```
Skill(skill="simplify")
```
- 触发条件：实现中有过度工程化的迹象
- 检查项：不必要的抽象、过度设计

**跳过条件**：改动极小（< 20 行），且风险可控

---

### 步骤 5：验证

**人格**：`yuki`（部署相关）或实现人格（功能验证）

**动作**：
- 运行测试套件
- 手动验证关键用户路径
- 检查是否有回归问题

**技能调用**：
```
Skill(skill="verify")
```
- 触发条件：功能完成后、上线前
- 用途：按验收标准逐条验证

---

## 进度跟踪

执行过程中更新 `tasks/feature-list.json`：
1. 步骤 1 完成 → 状态改为 `in_progress`
2. 步骤 3 完成 → 状态改为 `verifying`
3. 步骤 5 通过 → 状态改为 `passing`

更新 `tasks/session-handoff.md` 记录当前进度。

## 变体

- **纯前端功能**：跳过步骤 2（无架构变更）→ 步骤 3（仅 mio）→ 步骤 4 → 步骤 5
- **纯后端功能**：跳过步骤 2 → 步骤 3（仅 rei）→ 步骤 4 → 步骤 5
- **紧急修复**：步骤 1（快速确认范围）→ 步骤 3（仅修复）→ 步骤 5 → 步骤 4（轻量审查）
