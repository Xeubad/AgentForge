# PROJECT_STRUCTURE 模板

> **用途**：项目的完整架构快照，供 AI 阅读后作为开发模板依据。所有内容基于当前代码库实际实现，非推测。
> **位置**：项目根目录 `PROJECT_STRUCTURE.md`（不是 `tasks/` 下，因为它是项目权威文档，与 README 同级）。
> **生命周期**：跨会话持久化，结构变更时同步更新。
> **与 project-context.md 的关系**：
> - `project-context.md`（tasks/ 下）= 轻量问卷答案，回答「用什么技术、什么阶段、什么规范」
> - `PROJECT_STRUCTURE.md`（项目根）= 完整架构快照，回答「在这个项目里具体怎么写代码」
> - 两者可共存：`PROJECT_STRUCTURE.md` 是权威，`project-context.md` 是补充
>
> **AI 读取优先级**：`PROJECT_STRUCTURE.md` > `project-context.md` > 完整开局问卷
>
> **不适用场景**：脚本、单文件工具、纯配置仓库等无「架构」可谈的项目不需要本文件，用 `project-context.md` 即可。

---

## 使用说明

本文件是**模板**，实际项目应根据规模裁剪：

- **小项目（< 500 行）**：只填第 1、2、3、10 段
- **中等项目（500-5000 行）**：填第 1-6、8、10 段
- **大项目（> 5000 行）**：全部填写

**每段都可留空或删除**——留下的必须是真实的，别为了填满而编造。

**首次生成**：可以让 AI 用 `/generate-project-structure` 命令扫描代码自动生成初稿，再由你审核修正。

## 维护说明

- **谁写**：首次由 AI 通过 `/generate-project-structure` 扫描代码生成初稿，之后由你审核维护
- **何时更新**：
  - 新增 API / 表 / 模块 → 更新对应表
  - 关键设计决策拍板 → 追加到第 10 段
  - 发现已知问题 → 追加到第 9 段
  - 技术栈升级 → 更新第 2 段
- **可执行更新命令**：`/update-project-structure`（AI 会检查代码与本文件差异，给出更新建议）
- **不写的东西**：
  - 一次性任务进度（进 `tasks/`）
  - 临时 TODO（进 `tasks/todo.md`）
  - 短期 blocker（进 `session-handoff.md`）
