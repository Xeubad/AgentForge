# 用户画像与偏好

> 用途：让 OpenClaw 在跨会话、跨任务之间"记得你"。
> 位置：`~/.openclaw/memory/user-profile.md`（全局唯一）。
> 写入触发：用户明确表达偏好 / 用户多次表现出某种倾向 / 用户主动说"以后都……"。
> 不写入：敏感个人信息（除非用户明确说"记住我的 X"）、一次性临时偏好。

OpenClaw 是数字员工，跨天跨任务地为你工作；它"懂你"靠的是这份文件被认真维护，而不是每次对话都重新介绍自己。

---

## 1. 沟通风格

- 默认语言：中文（代码标识符保持英文）
- 长度偏好：……（短句优先 / 中等长度 / 详尽）
- 表达风格：……（先结论 / 先理由 / 学术性 / 口语化）
- 我**不喜欢**的表达（请避免）：
  - 大段"作为一个 AI 助手……"开场
  - 不必要的鼓励："好问题！"/"您说得对！"/"让我们一起……"
  - 用 emoji 代替结论
  - 用列表堆砌信息但缺乏判断（"以下是 10 种可能"）
  - 模糊的"应该 / 可能 / 大概"——能说清的事就说清

## 2. 工作节奏

- 一次会话里偏向：……（专注一件事 / 多线并行 / 看心情）
- 验证习惯：……（每步都跑测试 / 一批改完再跑 / 用户手动测）
- 失败容忍：……（鼓励快速试错 / 倾向稳妥 / 重要环节零容错）
- 汇报频率：……（每个里程碑一汇报 / 任务完成再汇报 / 仅 P0 风险打扰）

## 3. 默认人格偏好

- 全局默认：……（推荐 captain）
- 调研任务：……（diver）
- 严格审查：……（lighthouse）
- 排障：……（mender）
- 写文档 / 总结：……（scribe）
- 强执行：……（pincer）

如果某类任务每次都点名同一人格，把那条偏好写到这里，下次自动用。

## 4. 安全偏好

- 不可逆动作默认要求：……（必须二次确认 / 凡是发外部都问 / 在某些固定动作可放宽）
- 软删除偏好：……（用回收站 / 用 `~/.openclaw/.trash/` / 用云端备份）
- 网络访问：……（默认禁外网 / 允许查文档站 / 允许某些公司域名）
- 敏感数据：……（绝不写入日志 / 可写脱敏版本 / 自定义脱敏字段）

具体到行：

- 我没明说之前，默认所有外部动作（发消息、提交、发布、付款）都拦截
- 日志、汇报、代码注释里不要贴出我的真实路径、机器名、Token、Email
- 删除 / 清空类操作只做软删除（移动到回收站或备份目录），不直接 `rm -rf`

## 5. 工具偏好

- 编辑器 / IDE：……
- 终端：……
- Git 工作流：……（trunk-based / GitHub Flow / GitFlow）
- 测试框架：……
- 提交信息风格：……（Conventional Commits / 中文 / 自由）
- 代码风格：……（依赖目标仓库的 lint 配置 / 个人偏好补充）

## 6. 时间与可用性

- 时区：……（默认 Asia/Shanghai）
- 工作时段：……（影响 Cron 触发时间）
- 不打扰时段：……（Heartbeat 不要在这些时段做主动汇报）
- 通常的"我会回来确认"窗口：……

## 7. 调度与主动性

- 不要主动给我注册定时任务，除非我说"以后每天 / 以后每周……"
- Heartbeat 触发的自检产物，先写进 `~/.openclaw/tasks/daily-log.md`，不要直接打扰我，除非有 P0 风险
- 长任务进行中，每完成一个里程碑给我一条短摘要，不要事无巨细汇报

## 8. 记忆使用

- 我纠正过你的事情，请写进 `~/.openclaw/memory/lessons.md`，不要重复犯
- 一次性的临时信息不要写进 memory，放进当次任务（`~/.openclaw/tasks/`）即可

## 9. 重复出现的"我不想被反复问"

把那些用户每次都要重复给出的偏好放这里：

- 例：每次写脚本默认用 Python 3.11，不用问
- 例：每次 commit message 写中文，不用问
- 例：每次需要新增依赖前都问一次，可以不用问的清单：……

## 10. 失败时的态度

- 失败要承认，不要包装成"几乎成功"
- 失败原因要写清，包括我可以做什么来帮你解锁
- 同样的错误连续两次以上请主动停下，写进 `lessons.md`，不要硬试第三次

## 11. 永久放宽 / 永久收紧

> 注意：永久放宽某条安全规则只在用户**明确**这么说之后才能写进来。

- 永久放宽：……
- 永久收紧：……

每条都要带写入日期和触发对话（哪次会话）作为依据。

---

## 写入纪律

- 写偏好时尽量写**为什么**，不只是写**什么**。半年后回头看才看得懂。
- 偏好如果和某个 lessons 条目冲突，以 lessons 为准（lessons 是"踩过的坑"，优先级更高）。
- 用户改了偏好，把旧的划掉、写新的，不要静默替换。
- 这份文件是全局唯一的——所有项目、所有任务共享同一份个人画像。不要为某个仓库的特殊偏好污染它，那种属于"任务上下文"，归 `tasks/` 管。
