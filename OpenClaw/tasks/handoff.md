# 会话交接

> 用途：跨会话之间的"上一棒"。下一个 OpenClaw 会话开工的第一件事是读这份。
> 位置：`~/.openclaw/tasks/handoff.md`（全局唯一）。
> 写入：长会话结束时 / 上下文即将膨胀时 / 用户主动说"今天就到这里" / 大块任务做完一个里程碑。
> 写入人：scribe（必要时 captain 也能写）。
>
> OpenClaw 是常驻数字员工，交接条目可能涉及多个工作目录；请在每条交接里写明 **workdir**，避免下一会话误把 A 项目的状态用到 B 项目。

## 维护原则

- 不要让 handoff 变成日志。这里只保存"接得上"必需的信息，啰嗦的过程进 daily-log。
- 阻塞条目要写"解锁条件"，否则下一轮会重新猜测。
- 下一动作必须**动词开头**，不要写形容词。
- 完成的任务保留在历史交接中作为里程碑，不要清空，但单条不超过 20 行。
