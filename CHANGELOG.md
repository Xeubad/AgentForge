## v1.1 — 2026-06-26（跨子目录优化：收敛阶段增强 + 子代理协作 + cursor 现代化）

> 主线：针对用户「开发、运维、审查」三大工作场景，优化除 Claude Code 外的 4 个子目录。

### 变更（M 级）

- **M1 · CodeX 子代理协作优化**：`CodeX/AGENTS.md` 新增「子代理协作」段，定义 6 个 agent 的定位、调用时机和 4 种协作模式（单 agent / 接力 / 并行 / 审查）。`GLOBAL-AGENTS.md` 同步新增「子代理选择指南」。
- **M2 · HermesAgent 收敛阶段增强**：`HermesAgent/HERMES.md`「3.1 收敛」新增主动提问机制（5 个维度），「3.2 侦察」扩充为「先读、先问、不先写」。
- **M3 · OpenClaw 收敛阶段增强**：`OpenClaw/CLAW.md`「2.1 收敛」新增主动提问机制（5 个维度），「2.2 侦察」扩充为「先读、先问、不先写」。
- **M4 · cursor 规则现代化**：`cursor/Cursor 高效开发规则.md` 重写为 `.cursor/rules/*.mdc` 分层架构指南，明确 4 个 mdc 文件的职责和 globs 规则，标记旧 `.cursorrules` 为兼容入口。

### 变更（L 级）

- L1：`CodeX/` 和 `cursor/` 无 CHANGELOG.md，本次变更记录在根 CHANGELOG。
- L2：`人格提示词/` 无变更需求（基金经理人.md 已完成）。

### 验证（证据）

- `HermesAgent/HERMES.md` 收敛段新增 5 维度提问，侦察段扩充为 5 条
- `OpenClaw/CLAW.md` 收敛段新增 5 维度提问，侦察段扩充为 5 条

### 未验证项（风险）

## v1.0 — 2026-06-14（建立根级 CLAUDE.md 与 CHANGELOG）

> 主线：把本仓库从"6 个并列子目录"升级为"有路由层的配置模板合集"，让进入仓库的 AI 能快速定位到正确子目录。

### 变更（H 级）

- **H1 · 新建根 `CLAUDE.md`**：建立仓库级路由层。明确仓库定位为"6 套面向不同 AI 工具的高效率配置方案合集"，目标读者仅 AI。包含子目录速查表、共同骨架、各子目录差异化要点、任务路由、维护原则、输出评估维度六个核心段。
- **H2 · 人格机制按现状描述不强行统一**：Claude Code 标记为"唯一具备人格自动路由"的子目录；Codex / Hermes / OpenClaw 各自的人格机制按现状描述；Cursor 标"无人格机制"；"人格提示词/" 定位为"独立于 AI 软件配置的第三方散件人格库"，明确不要塞进其他工具的人格配置。

### 变更（M 级）

- **M1 · 新建根 `CHANGELOG.md`**：建立仓库级变更日志，与子目录 CHANGELOG 分层维护。写入边界限定为四类——子目录增删、根 CLAUDE.md 变更、跨子目录同步性调整、共同骨架调整。
- **M2 · 根 `CLAUDE.md` 新增「仓库变更日志（强制）」段**：明确根 CHANGELOG 的写入边界、不该记的内容、统一格式（与子目录 CHANGELOG 对齐）和 H/M/L 分级判定标准。

### 验证（证据）

- 仓库根目录新增两个文件：`CLAUDE.md`（路由层）、`CHANGELOG.md`（本文件）。
- 子目录全部未动：`Claude Code/` / `CodeX/` / `HermesAgent/` / `OpenClaw/` / `cursor/` / `人格提示词/` 内文件保持原状。
- 子目录速查表 6 行对齐实际目录：每个子目录的入口文件、部署形态、人格机制按其 README 实际描述填入。

### 未验证项（风险）

- 路由表的实际有效性需要在新会话中验证——进入仓库后是否能按"用户意图 → 去哪"映射准确路由到对应子目录。下一步：开新会话，给出"我想改 Hermes 的 SOUL 配置"类指令，观察是否直接进入 `HermesAgent/` 而不绕路。
- 「共同骨架」段落与各子目录内已有规则可能存在表述差异；当前未做跨子目录措辞统一审计。下一步：必要时做一次跨子目录的"先读后改"等核心原则措辞比对。
