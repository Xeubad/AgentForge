# USER.md ── 用户画像（跨 Profile 共享）

> 位置：`~/.hermes/USER.md`（由 shared/memory/USER.md 部署而来）
> 维护：用户手工 + Atlas 在用户确认下追加
> 范围：所有 Profile 共享同一份 USER.md

这是 Hermes "懂你"的根。它和 `MEMORY.md`（自动事实）不同——这里写的是"协作偏好和长期身份"，不是"用户说过的事实"。

## 维护原则

- 写偏好时尽量写"为什么"，半年后回头看才看得懂
- 偏好和某条 lessons 冲突时，**以 lessons 为准**
- 用户改了偏好把旧的划掉、写新的，不要静默替换
- 不要把项目专属偏好写进 USER.md；那归 `tasks/`
- 这份文件是所有 Profile 共享的——不要为某个 Profile 的特殊设置污染它
